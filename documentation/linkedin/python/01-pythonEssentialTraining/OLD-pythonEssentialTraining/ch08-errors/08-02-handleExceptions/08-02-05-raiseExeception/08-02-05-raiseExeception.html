08-02-05- raise execeptions

 We can re-use exceptions on multiple functions. Use the @handleExceptions decorator from the previos lesson to raise exceptions 

def handleExceptions(func):
    def wrapper():
        try:
            func()
        except TypeError:
            print('type error')
        except ZeroDivisionError:
            print('zero division')
        except Exception:
            print('error')
    return wrapper

raise allows use to call specifically the except Exception within the handleExceptions function 

@handleExceptions
def raiseError():
    raise Exception()

Pass arguments into decorators

If we try to add variables within the raiseError() function to work with, we'll get a type error

We must pass arguments into the wrapper() function within the handleExceptions() function by using *args 

def handleExceptions(func):
    def wrapper(*args):
        try:
            func(*args)
        except TypeError:
            print('type error')
        except ZeroDivisionError:
            print('zero division')
        except Exception:
            print('error')
    return wrapper


@handleExceptions
def raiseError(n):
    if n == 0:
        raise Exception()
    print(n)


raiseError(1)

raiseError(0)
