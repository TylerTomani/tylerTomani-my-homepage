07-01- The anatomy of Classes

Instance attributes are named so because any new instance of the class made automatically has these attributes.

We can see this by creating new variables and instantiating them with the Dog class. 

class Dog:
    def __init__(self,name):
        self.name = name
        self.legs = 4
    def speak(self):
        print(self.name + ' say Bark')
myDog = Dog('Rover')
print(myDog.name)
print(myDog.legs)

These instance attributes are hardcoded So, we can not see them or change them on the Dog class itself

static attributes

 To see and change the instance attribute we must make it an attribute within the class itself. These attributes are refered to as static attributes
 Do this by assigning the legs variable outside of the __init__ function. Each instance of the class still retains the value of legs but now, we can call Dog.legs 

class Dog:
    legs = 4
    def __init__(self,name):
        self.name = name
myDog = Dog('Rover')
print(myDog.name)
print(myDog.legs)
print(Dog.legs)

  static attributes are unchanging with each instance. These are meant to be assiged to constant values, but Be Careful because you can change them.

 If we reassign Dog.legs to 3 and create a new instance, you will see the static legs attribute has been changed 

Dog.legs = 3
newDog = Dog('Scruffy')
print(newDog.name)
print(newDog.legs)

It is good practice to prefix a static attribue with a _
_legs. This warns anyone working with the code that you should change this attribute with caution
