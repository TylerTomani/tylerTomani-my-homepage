05-02-charClassesQuantifiers

Adding a ^ character before a class or quantier does the opposite

[] Character Class

Matches a single character within the []

  • /a[xyz]b/ matches:
    • axb
    • ayb
    • azb
  • /a[xyz]b/ does not match axyzb
  • /a[a-z]c/ matches any lowercase character a-z
  • /a[A-Z]c/ matches any uppercase character a-z
  • /a[a-zA-Z]c/ matches any lower or uppercase character a-z
  • /a[0-9]/ any digit 1-9
  • /a[^0-9]/ would search for anything NOT a number
  • /a[^a-z]c/ would search for anything NOT lowercase
    • "aBc

* matches zero or more repeats

Matches 0 or more occurances of the previous item

  • /ab*c/ matches abbc
  • /ab*c/ matches abbbbbbc
  • /ab*c/ matches ac
    • b/c the b does NOT need to be there

+ matches one or more occurances of previous item

  • /ab+c/ would not match ac
    • b/c the b needs to be there at least once

? matches ZERO or ONE occurance

  • /ab?c/ matches ac
  • /ab?c/ matches abc
  • /ab?c/ does not match abbc

{n} Specify an exact number of repeats

  • /ab{2}c/ matches abbc

{n,} means whatever n is or more

  • /ab{2,}c matches abbc or abbbc etc

{n,m} min max repeates

  • /ab{2,4}c/ matches abbc, abbbc abbbc

 An item is usually one character but can be a string of characters

  • /ab+c/ matches one or more abbc
  • /(ab)+c matches abababababc

Greediness - HTML example

Matching an html tag

 The following line will match the entire tag becuase regex quantifiers are Greedy
This means it will extand to the last closing > tag.

  • /<.+> will match the entire line <i>italic tag</i>

If you only want it to match the opening tag:

  • /<[^>]+>/ matches only <i>

This will match the opening < then any amount of anything that is NOT a closing >,
